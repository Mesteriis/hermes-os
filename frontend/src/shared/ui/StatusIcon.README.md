# StatusIcon

## Description
Semantic status icon primitive.

## When to use
Use for idle, active, success, warning, danger, offline and syncing states.

## When not to use
Do not replace textual status on critical flows.

## Accessibility
The wrapper exposes an accessible image label.

## Keyboard
No keyboard behavior.

## Examples
`<StatusIcon status="success" label="Synced" />`

## Anti Patterns
Do not communicate risk with color alone.
