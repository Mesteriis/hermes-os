# ComingSoonState

## Description
Reserved-surface state for future UI contracts.

## When to use
Use in Storybook or internal scaffolding where a surface is intentionally unavailable.

## When not to use
Do not ship it as a replacement for a required implemented workflow.

## Accessibility
Provides readable heading and description.

## Keyboard
No custom keyboard behavior is added.

## Examples
`<ComingSoonState title="Inspector actions" />`

## Anti Patterns
Do not use this to fake completion of product behavior.
