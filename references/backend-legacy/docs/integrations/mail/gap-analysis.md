# Mail Domain Gap Analysis

Status date: 2026-07-11.

This document tracks the requested production mail-platform scope against the
current repository state. It uses the required labels:

- `IMPLEMENTED` — implemented and covered by current code/tests or documented
  implementation.
- `PARTIAL` — meaningful implementation exists, but production scope is not
  complete.
- `BROKEN` — implementation exists but current evidence shows it does not work.
- `MISSING` — no durable implementation found in the current repository.
- `REGRESSION` — current behavior is worse than previously documented behavior.

Evidence sources: `docs/integrations/mail/*.md`, `backend/src/domains/communications/`,
`backend/src/platform/events_api/`, `frontend/src/domains/communications/` and
current validation results.

## Closure Audit

Run `make mail-domain-closure-audit` to report current row-level closure state.
`make mail-domain-closure-gate` fails until every tracked row is `IMPLEMENTED`
or explicitly `EXCLUDED` by an accepted ADR; percentages are not readiness
evidence.

| Attachment Safety Scan | PARTIAL | Conservative heuristic prefilter marks executable magic bytes, active-content extensions, macro-enabled Office extensions and MIME/extension mismatches as `malicious` or `suspicious`. It performs bounded central-directory inspection of OOXML Office containers (50 MiB / 10,000 entries), and bounded in-memory CFB/OLE inspection of header/FAT/directory metadata with sector and cycle limits: `vbaProject.bin`/`vbaData.xml` and legacy `VBA`/`_VBA_PROJECT_CUR`/`Macros` storage markers are `suspicious`; malformed, over-limit or uninspected legacy containers remain quarantined without opening content streams. Mail sync and local imports use a timeout-bounded ClamAV `INSTREAM` client when `HERMES_CLAMAV_ADDR` is configured; only `OK` becomes `clean`, malware signatures become `malicious`, and unavailable/malformed scanner responses remain quarantined as `not_scanned`. A bounded 60-second local rescan worker revisits up to 50 such incoming and staged-for-send attachments once ClamAV recovers; it rereads only local blobs, verifies the recorded byte size and SHA-256 before scanning, and conditionally persists a verdict only while the attachment still remains `not_scanned` for that immutable blob. Preview and ZIP inspection require a `clean` verdict, so quarantined bytes cannot be opened through REST or ConnectRPC. PDF/DOCX safe preview rendering produces a bounded derived PNG blob; the durable artifact records its renderer and source SHA-256, and is refused if the source blob changes, its storage/content type is invalid or its PNG signature/size check fails. Clean PDF attachments are rasterized into a separate image-only PDF CDR artifact; clean DOCX attachments are parsed only through bounded ZIP/XML text extraction and rebuilt into the same image-only PDF form. Both are limited to four pages/2 MiB, leave the original immutable, and REST download rechecks the current scan verdict, source SHA-256, local storage, content type, size and PDF trailer. Successful ZIP inspection reports are stored as versioned scan metadata bound to the current blob SHA-256; stale reports are ignored after replacement, while traversal, encryption, nested archives and configured size/depth/count limits are rejected without extraction. The Docker `attachment-extractor` runs PDF/DOCX/OCR parsers on a read-only blob mount in an internal network with no external route, dropped capabilities, read-only root filesystem and CPU/memory/process/temp-storage bounds; its loopback gateway carries only bounded framed requests. Attachment-based Telegram and WhatsApp media sends, including the WhatsApp executor's live/fixture byte-read path, require a matching `clean` `attachment_id`; a bare blob reference cannot bypass quarantine and policy/identity/scan mismatches dead-letter without retry. Attachment scan-status transitions append a privacy-safe `communication.attachment.processing_changed.v1` event atomically with the durable verdict; identical re-upserts emit no duplicate event. Outbox delivery fails permanently unless every linked attachment is clean, then verifies local blob size and SHA-256 before MIME assembly. Archive CDR, bounded nested archive extraction and full OLE stream analysis remain incomplete. |
