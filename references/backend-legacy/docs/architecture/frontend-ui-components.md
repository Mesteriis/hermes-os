## Component ownership

All components live under:

```text
frontend/src/shared/ui
```

Exports live in:

```text
frontend/src/shared/ui/index.ts
```

Styles live in:

```text
frontend/src/shared/ui/styles
```

## Component API policy

Components expose Hermes API, not vendor API.

Good:

```vue
<Button variant="outline" icon="tabler:search">Search</Button>
```

Bad:

```vue
<DropdownMenuRoot>
  <DropdownMenuTrigger />
</DropdownMenuRoot>
```

The second example belongs inside `shared/ui`, never in domains.

## Component styling policy

Components use stable class names:

```text
hermes-btn
hermes-input
hermes-dialog-content
hermes-dropdown-item
```

Class names are intentionally not Tailwind-heavy in templates. Tailwind can still be used in stories or outer layout, but UI Kit components should be restyled through CSS files and tokens.

## Objective completion scope

The explicit named component objective now has no remaining component names:

```text
Communication primitives: implemented.
Utility primitives: implemented.
```

Communication primitives from the objective remain UI-only requirements. They
must not import Telegram, Mail, WhatsApp, provider runtime, domains, stores or
API clients.

`HtmlPreview` is a generic media/viewer primitive for already-prepared HTML or
plain-text body fragments, including email-like content. It must stay provider
neutral, must not import Mail or Communications domains, and must make the
sanitization boundary explicit before rendering HTML.

`RichTextEditor` is a generic authoring primitive for context notes, local drafts
and evidence snippets. It emits sanitized HTML, uses a compact semantic toolbar
instead of document-processor controls, and must stay provider/domain neutral.

## Adding a new component

1. Add component under `frontend/src/shared/ui`.
2. Export from `frontend/src/shared/ui/index.ts`.
3. Add CSS to the smallest matching file under `styles/`.
4. Add Storybook story under `frontend/stories/ui`.
5. Add per-component documentation next to the component.
6. Add a targeted boundary or unit test for the component slice.
7. Run visual snapshot update if Storybook output changed.
8. Do not add domain logic.
9. Do not add API calls.
10. Do not import a domain store.
