### Summary / Резюме

Создаётся (или обновляется) страница `operations/documentation-map.md` в русской wiki Hermes. Она централизует ссылки на отчёты по тестовой производительности из директории `reports/test-performance/`, описывает их назначение, фиксирует безопасные точки входа для запуска тестов и архитектурные ограничения, зафиксированные в этих отчётах. Это устраняет ситуацию, когда актуальные измерения и договорённости о тестовом харнессе разбросаны по нескольким файлам без единого навигационного документа.

```markdown
## Точки входа для запуска тестов

Из базового замера и аудита Testcontainers:

- **Безопасные точки входа**: `make backend-test`, `make backend-validate`, `make test-unit`.
- Полный прогон backend-тестов **обязан** идти через сессионный харнесс `hermes_test_session` (`crates/testkit/src/bin/hermes_test_session.rs`).
- Прямой вызов `cargo test` на всём backend **не допускается** — он обходит контракт общей сессии с контейнерами и может оставлять «мусорные» Docker-контейнеры после прерываний или падений.
- Отчёты nextest генерируются скриптом `scripts/test/analyze-nextest-junit.mjs` (записывает результат в `reports/test-performance/` при явных прогонах).

## Архитектурные заметки

- **PostgreSQL** — основное persistent-хранилище.
- Docker-инфраструктура разработки находится в `docker/` (согласно ADR-0032; содержимое ADR не встроено в этот контекст, достоверность ссылки не подтверждена).
- Полный прогон backend-тестов должен выполняться **исключительно** через харнесс `crates/testkit`.
- **Redis** не вводится в тестовый стек без явного архитектурного решения и реальной потребности конкретной подсистемы Hermes.
```

### Drift candidates / Кандидаты на drift

В предоставленном контексте видны следующие потенциальные расхождения:

1. **Ссылка на `docs/development/testing/status.md`** (из `reports/test-performance/README.md`) — файл не встроен в контекст, его наличие и актуальность не могут быть подтверждены. Если файл отсутствует или устарел, навигация из документационной карты будет нарушена.
2. **Упоминание скриптов `scripts/test/run-nextest.sh`, `run-llvm-cov.sh`, `analyze-nextest-junit.mjs`** — ни один из этих скриптов не предоставлен в source‑pack, поэтому факт их существования и соответствия описанному поведению не верифицируем.
3. **Упоминание ADR‑0032** (из `2026-06-23-baseline.md`) — содержимое ADR отсутствует, проверить актуальность и релевантность решения невозможно.
4. **Упоминание `Makefile`, `crates/testkit`** — сами файлы не встроены; расхождений между отчётами не наблюдается, но актуальное состояние кода может отличаться от зафиксированных в отчётах дат (2026‑06‑23 и 2026‑06‑28).
5. **Наличие файлов `unit.json` и `backend-full.json`** (описаны в README) — не включены в контекст; их отсутствие или отличие от `.md`‑версий может свидетельствовать о незавершённой генерации.

Других расхождений между встроенными исходными файлами не обнаружено.
