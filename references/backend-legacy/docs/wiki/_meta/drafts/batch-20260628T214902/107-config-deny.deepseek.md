### Summary / Резюме

В wiki следует добавить описание конфигурационного файла `deny.toml`, используемого для автоматической проверки зависимостей проекта. Страница `operations/configuration.md` должна содержать перечень секций конфигурации с их значениями и комментарии на русском языке, основанные исключительно на содержимом файла.

### Proposed pages / Предлагаемые страницы

**`operations/configuration.md`**

```markdown
# Конфигурация операций

## deny.toml — проверка зависимостей

Файл `deny.toml` (корень репозитория) задаёт правила автоматической проверки используемых зависимостей.

### Секция `[graph]`

- `all-features = true` — при построении графа зависимостей учитываются все возможности (features) крейтов.

### Секция `[advisories]`

- `yanked = "warn"` — предупреждать, если в зависимостях обнаружены yanked-крейты.

### Секция `[licenses]`

- `allow` — список разрешённых лицензий для зависимостей:
  - `Apache-2.0`
  - `BSD-2-Clause`
  - `BSD-3-Clause`
  - `CDLA-Permissive-2.0`
  - `ISC`
  - `MIT`
  - `MPL-2.0`
  - `Unicode-3.0`
  - `Zlib`
- `confidence-threshold = 0.8` — минимальный порог уверенности при определении лицензии зависимости (значение 0.8).

### Секция `[bans]`

- `multiple-versions = "warn"` — предупреждать, если в графе присутствует несколько версий одной и той же зависимости.
- `wildcards = "allow"` — разрешено использование wildcard-версий при указании зависимостей.

### Секция `[sources]`

- `unknown-registry = "deny"` — запрещать зависимости из неизвестных реестров (регистрация отклоняется).
- `unknown-git = "warn"` — предупреждать при использовании зависимостей из неизвестных git-репозиториев.
```
