# Конфигурация Cargo

## Cargo.toml (рабочее пространство)

Файл `Cargo.toml` в корне репозитория определяет Cargo workspace.

```toml
[workspace]
resolver = "2"
members = ["backend", "crates/testkit"]
exclude = ["frontend/src-tauri"]
```

- **resolver**: `"2"` (разрешитель зависимостей версии 2).
- **members**: пакеты `backend` и `crates/testkit` включены в рабочее пространство.
- **exclude**: пакет `frontend/src-tauri` исключён из рабочего пространства.

## Профили сборки

Файл `Cargo.toml` задаёт профили `dev` и `test`.

```toml
[profile.dev]
debug = "line-tables-only"

[profile.test]
debug = "line-tables-only"
incremental = false
```

- **profile.dev**: уровень отладочной информации — `"line-tables-only"` (только таблицы строк).
- **profile.test**: уровень отладочной информации — `"line-tables-only"`; инкрементальная компиляция отключена (`incremental = false`).

## Cargo.lock

Файл `Cargo.lock` генерируется автоматически (`# This file is automatically @generated by Cargo`) и **не предназначен для ручного редактирования**. Он содержит зафиксированные версии и контрольные суммы всех зависимостей проекта.

В доступном фрагменте `Cargo.lock` видны, среди прочих, крейты:

- `axum`, `tokio`, `serde`, `serde_json`
- `bollard`, `async-nats`
- `argon2`, `blake2`
- `time`, `chrono`
- `hyper`, `tonic`, `prost`
- `rustls`, `native-tls`

Список неполный; полный перечень находится в `Cargo.lock`.
```
