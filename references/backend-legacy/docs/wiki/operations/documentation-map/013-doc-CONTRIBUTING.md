## CONTRIBUTING.md

- **Путь в репозитории:** `CONTRIBUTING.md`
- **Размер:** 1361 байт
- **Назначение:** руководство по участию в разработке Hermes Hub — локальной
  персональной системе памяти для коммуникаций, знаний, отношений, проектов,
  документов, решений, обязательств и контекста.
- **Ключевые разделы:**

### Перед началом работы

- Прочитать `AGENTS.md` и актуальные ADR в `docs/adr/`.
- Не добавлять облачные зависимости, операции записи у провайдеров или
  побочные эффекты агентов ИИ без архитектурного обоснования через ADR.
- Не коммитить секреты, приватные данные почты/сообщений, локальные фикстуры с
  персональным содержимым, `docker/.env` или сгенерированные данные из
  `docker/data/`.
- Изменения должны быть небольшими, тестируемыми и соответствовать структуре
  Rust, SvelteKit и Tauri.

### Настройка окружения для разработки

```sh
make docker-env
make validate
```

Только backend:

```sh
make backend-validate
```

Только frontend:

```sh
make frontend-check
make frontend-build
```

### Pull Request'ы

- Объяснять, что изменено и почему.
- Ссылаться на ADR или документы, ограничивающие изменение.
- Приводить команды валидации и их результаты.
- Добавлять регрессионные тесты при изменении поведения.
- Исключать сгенерированные/локальные данные из диффа.

### Безопасность и приватность

- Импортированные сообщения, документы и данные провайдеров считаются частными
  и недоверенными.
- Не включать приватные пользовательские данные в issues, pull request'ы,
  скриншоты или тестовые фикстуры.
```
