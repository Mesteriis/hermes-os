```markdown
# Документация безопасности

Проект Hermes Hub обрабатывает личные коммуникации, документы, ссылки на учётные данные и локальные AI-процессы. О проблемах безопасности необходимо сообщать конфиденциально.

## Поддерживаемые версии

Проект находится на предрелизной стадии. Исправления безопасности применяются к ветке по умолчанию.

## Сообщение о уязвимости

**Не создавайте публичные issues для отчётов, связанных с безопасностью.**

Отправляйте сообщение мейнтейнеру по электронной почте или используйте приватный механизм отчётов GitHub (если он включён для репозитория). В отчёте укажите:

- затронутый коммит или версию;
- характер воздействия и затронутые данные или возможности;
- шаги для воспроизведения;
- были ли раскрыты секреты, приватные сообщения или документы.

## Ожидания по безопасности

- **Никогда** не фиксируйте секреты, токены провайдеров, пароли приложений, приватные ключи или локальные файлы `.env`.
- **Никогда** не публикуйте тела приватных сообщений, содержимое документов или сырые записи провайдеров в issues или тестовых фикстурах.
- Держите запись провайдера, деструктивные действия и автоматизацию за явными границами возможностей, подтверждения и аудита.
- Обрабатывайте импортированные документы и сообщения как ненадёжный ввод.
```
