# Mail Domain Gap Analysis

Status date: 2026-06-15.

This document tracks the requested production mail-platform scope against the
current repository state. It uses the required labels:

- `IMPLEMENTED` — implemented and covered by current code/tests or documented
  implementation.
- `PARTIAL` — meaningful implementation exists, but production scope is not
  complete.
- `BROKEN` — implementation exists but current evidence shows it does not work.
- `MISSING` — no durable implementation found in the current repository.
- `REGRESSION` — current behavior is worse than previously documented behavior.

Evidence sources: `docs/mail/*.md`, `backend/src/domains/communications/`,
`backend/src/platform/events_api/`, `frontend/src/domains/communications/` and
current validation results.

## Current Next Implementation Candidates

1. Outbox reliability: connect external provider webhook/subscription sources to
   the structured provider event path and expand delivery UX beyond the compact
   status strip.
2. Attachment translation and preview: OCR/extracted-text pipeline, safe preview
   rendering and translation of extracted attachment text.
3. Provider-side bulk actions: queued Gmail/IMAP batch mutations with progress
   events and replayable status.
4. Folder/provider parity and advanced search UX: provider-folder
   synchronization, richer multi-parent or provider-mapped folder management,
   and deeper saved-search execution validation beyond the current local
   hierarchy-aware Folder Hierarchy and nested-group Rules Builder flows.
5. Typed realtime mail events: event contracts for message/draft/search changes
   and entity-level patch streaming.
