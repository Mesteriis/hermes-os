# Phase 2 Domain Ownership Audit (2026-06-18)

Scope: Phase 2 ownership audit for active domain, engine and workflow
documentation. This file intentionally does not move docs, rename code, rewrite
ADR bodies, create new domains, close open questions or propose schema changes.

Owner decisions applied from the 2026-06-18 audit thread:

- ADR-0093 is canonical; SvelteKit is historical or migration context only.
- ADR status policy should expand to Draft, Proposed, Accepted, Implemented,
  Needs Review, Superseded, Deprecated, Historical and Rejected.
- ADR-0075 remains a Reserved / Missing gap, not a stub ADR.
- Active documentation uses Personas; code-level `persons` remains a
  compatibility layer.
- Communications owns Email, Telegram, WhatsApp, Calls and Meetings as
  channels.
- Knowledge is an emergent layer, not a standalone domain for now.
- Event vocabulary must split at least into Domain Event, Calendar Event and
  Timeline Event; Integration Event remains a candidate term.
- Implementation docs should live under `docs/implementation/`.
- Historical plans, reviews and closure roadmaps should move to `docs/archive/`.
- `.DS_Store` files should be removed; this pass removed the two untracked
  files found under docs.

## Restructuring Gate

Do not move, rename or merge documentation trees until the following are true:

1. ADR status policy is documented.
2. Documentation governance defines Canonical, Implementation, Historical and
   Compatibility document types.
3. Event taxonomy is added to the glossary/foundation docs.
4. Knowledge item and generic Memory record ownership are clarified.
5. Observation persistence policy is clarified enough to avoid accidental
   `observations` as a god-domain.
6. Active docs are patched so Relationship ownership points to Relationships,
   not Knowledge Graph.

After those gates, the channel and archive moves are lower-risk mechanical
work because the bounded contexts are explicit.
